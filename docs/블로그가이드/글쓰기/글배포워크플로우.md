# 글 배포 워크플로우

## 전체 순서

```
1. 글 파일 생성
        ↓
2. 내용 작성 (마크다운)
        ↓
3. 로컬 서버로 미리보기
        ↓
4. draft: false 로 변경
        ↓
5. git add → git commit → git push
        ↓
6. GitHub Actions 자동 빌드·배포 (1~2분 소요)
        ↓
7. https://gjwoo1996.github.io 에서 확인
```

---

## 단계별 명령어

### 1단계: 새 글 생성

```bash
hugo new posts/my-post.md
```

### 2단계: 내용 작성

에디터(VS Code 등)로 `content/posts/my-post.md`를 열어 글을 작성합니다.

### 3단계: 로컬 미리보기

```bash
hugo server -D
# http://localhost:1313 에서 확인
```

### 4단계: draft 해제

글이 완성되면 Front Matter에서 `draft: true` → `draft: false`로 변경합니다.

### 5단계: GitHub에 올리기

```bash
git add .
git commit -m "글 추가: 글 제목"
git push
```

### 6단계: 배포 확인

GitHub 저장소 → `Actions` 탭에서 빌드/배포 진행 상황을 확인할 수 있습니다.  
초록색 체크(✓)가 뜨면 배포 완료입니다.

---

## git commit 메시지 작성 가이드

```bash
# 새 글 추가
git commit -m "글 추가: Spring Boot 입문기"

# 기존 글 수정
git commit -m "글 수정: hello-world 오탈자 교정"

# 설정 변경
git commit -m "설정: 소셜 아이콘 LinkedIn 추가"

# 이미지 추가
git commit -m "이미지 추가: profile 사진"
```

---

## 배포 없이 특정 글만 숨기고 싶을 때

`draft: true`로 변경 후 push하면 다음 빌드부터 해당 글이 사이트에서 제외됩니다.

```yaml
---
title: "임시로 숨길 글"
draft: true   # 이렇게 변경 후 git push
---
```
