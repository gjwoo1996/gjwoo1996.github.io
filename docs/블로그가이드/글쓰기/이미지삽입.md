# 이미지 삽입

## 이미지 저장 위치

이미지는 반드시 `static/` 폴더 안에 저장해야 합니다.

```
static/
└── images/
    ├── profile.png
    ├── hello-world-cover.png
    └── screenshot.jpg
```

`static/` 폴더에 있는 파일들은 빌드 시 루트(`/`)로 복사됩니다.

| 저장 경로 | 접근 URL |
|-----------|---------|
| `static/images/profile.png` | `/images/profile.png` |
| `static/favicon.ico` | `/favicon.ico` |

---

## 글 안에서 이미지 사용하기

```markdown
![이미지 설명](/images/profile.png)
```

형식: `![대체 텍스트](이미지 경로)`

- **대체 텍스트**: 이미지가 로드되지 않을 때 표시되는 텍스트. SEO에도 영향을 줍니다.
- **이미지 경로**: `/`로 시작하는 절대 경로 사용을 권장합니다.

---

## 글 썸네일(커버 이미지) 설정

Front Matter에 `cover` 항목을 추가하면 목록 페이지와 글 상단에 썸네일이 표시됩니다.

```yaml
---
title: "글 제목"
date: 2026-05-03T10:00:00+09:00
draft: false
cover:
  image: "/images/my-cover.png"    # 커버 이미지 경로
  alt: "커버 이미지 설명"            # 대체 텍스트
  caption: "이미지 출처: ..."        # 이미지 아래 캡션 (선택)
---
```

## 파비콘(브라우저 탭 아이콘) 추가

파비콘이 없으면 브라우저 탭에 기본 아이콘이 표시됩니다.

1. `favicon.ico` 파일 준비 (32x32 픽셀 권장)
2. `static/` 폴더에 저장

```
static/
├── favicon.ico
└── favicon-16x16.png   (선택)
└── favicon-32x32.png   (선택)
```

별도의 설정 없이 자동으로 인식됩니다.

---

## 주의사항

- 이미지를 `content/posts/` 안에 저장하면 경로가 꼬일 수 있습니다. **반드시 `static/` 폴더에 저장**하세요.
- 이미지 파일명에 한글이나 공백을 사용하면 URL 인코딩 문제가 생길 수 있습니다. 영문 소문자와 하이픈(`-`) 사용을 권장합니다.
  - 좋은 예: `my-profile-photo.png`
  - 나쁜 예: `내 프로필 사진.png`
