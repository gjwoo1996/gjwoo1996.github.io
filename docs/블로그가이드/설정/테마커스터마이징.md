# 테마 커스터마이징

## 핵심 원칙: 테마 파일을 직접 수정하지 않는다

`themes/PaperMod/` 폴더 안의 파일은 **절대 직접 수정하지 않습니다.**

이유:
- PaperMod는 Git Submodule로 연결된 외부 저장소입니다.
- `git submodule update` 명령으로 테마를 업데이트하면 수정 사항이 사라집니다.

대신, 루트의 `layouts/`, `assets/`, `static/` 폴더에 **같은 경로의 파일을 만들면**  
Hugo가 테마 파일 대신 내 파일을 우선 적용합니다.

```
테마 파일:  themes/PaperMod/layouts/partials/footer.html
내 파일:    layouts/partials/footer.html   ← 이 파일이 우선 적용됨
```

## 커스텀 CSS 추가하기

가장 자주 쓰는 커스터마이징 방법입니다.

1. 아래 경로에 CSS 파일을 생성합니다.

```
assets/css/extended/custom.css
```

PaperMod는 이 경로를 자동으로 불러오도록 설계되어 있습니다.  
`extended/` 폴더 안의 모든 `.css` 파일이 자동으로 포함됩니다.

2. 원하는 스타일을 작성합니다.

```css
/* 본문 최대 너비 조정 */
.post-content {
  max-width: 800px;
}

/* 링크 색상 변경 */
.post-content a {
  color: #0066cc;
}

/* 코드 블록 폰트 크기 */
.post-content pre {
  font-size: 14px;
}
```

## 특정 HTML 부분 덮어쓰기

PaperMod의 특정 레이아웃을 수정하고 싶을 때 사용합니다.

### 예시: footer 수정

```bash
# 1. 원본 파일 위치 확인
themes/PaperMod/layouts/partials/footer.html

# 2. 같은 경로로 내 파일 생성 (원본 내용 복사 후 수정)
layouts/partials/footer.html
```

### 예시: 홈 화면 레이아웃 수정

```bash
# 원본
themes/PaperMod/layouts/index.html

# 내 파일 (우선 적용됨)
layouts/index.html
```

## 다국어 UI 텍스트 변경 (i18n)

PaperMod의 일부 UI 텍스트가 영어로 표시될 때 사용합니다.

1. `i18n/ko.yaml` 파일 생성

```yaml
# i18n/ko.yaml
- id: read_time
  translation: "분 읽기"

- id: words
  translation: "단어"

- id: toc
  translation: "목차"
```

PaperMod의 원본 번역 파일은 `themes/PaperMod/i18n/` 폴더에 있습니다.  
덮어쓰고 싶은 항목만 내 `i18n/ko.yaml`에 추가하면 됩니다.

## 테마 업데이트 방법

```bash
# PaperMod 최신 버전으로 업데이트
git submodule update --remote --merge

# 변경사항 커밋
git add themes/PaperMod
git commit -m "테마 업데이트: PaperMod 최신 버전"
git push
```
