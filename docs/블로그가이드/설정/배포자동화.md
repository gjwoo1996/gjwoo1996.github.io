# 배포 자동화 (GitHub Actions)

## 개요

`.github/workflows/hugo.yml` 파일이 자동 배포를 담당합니다.  
`git push`만 하면 이 파일이 실행되어 빌드와 배포를 자동으로 처리합니다.

---

## 워크플로우 파일 전체 설명

```yaml
name: Deploy Hugo site to GitHub Pages

# 언제 실행할지 정의
on:
  push:
    branches: ["main"]   # main 브랜치에 push될 때마다 실행
  workflow_dispatch:     # GitHub 웹에서 수동으로도 실행 가능

# 필요한 권한 설정
permissions:
  contents: read   # 저장소 내용 읽기
  pages: write     # GitHub Pages에 쓰기
  id-token: write  # 배포 인증 토큰

# 동시 실행 제어
concurrency:
  group: "pages"
  cancel-in-progress: false  # 이미 배포 중이면 중단하지 않음
```

---

## build 작업 (빌드 단계)

```yaml
jobs:
  build:
    runs-on: ubuntu-latest   # GitHub 서버(Ubuntu)에서 실행

    steps:
      # 1. 저장소 코드 다운로드
      - name: Checkout
        uses: actions/checkout@v4
        with:
          submodules: recursive  # PaperMod 서브모듈도 함께 받음
          fetch-depth: 0         # 전체 git 기록 받기
                                 # (Hugo의 lastmod 기능에 필요)

      # 2. Hugo 설치
      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v3
        with:
          hugo-version: "0.146.0"  # Hugo 버전 고정
          extended: true           # SCSS 등 확장 기능 포함 버전

      # 3. Pages 설정 정보 가져오기
      - name: Setup Pages
        id: pages
        uses: actions/configure-pages@v5

      # 4. Hugo 빌드 실행
      - name: Build with Hugo
        env:
          HUGO_ENVIRONMENT: production
        run: hugo --minify --baseURL "${{ steps.pages.outputs.base_url }}/"
        # --minify: HTML/CSS/JS 파일 용량 최소화 (빠른 로딩)
        # --baseURL: GitHub Pages 실제 주소를 자동으로 주입

      # 5. 빌드 결과(public/ 폴더) 업로드
      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: ./public
```

---

## deploy 작업 (배포 단계)

```yaml
  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}  # 배포된 URL

    runs-on: ubuntu-latest
    needs: build   # build 작업이 성공해야 실행됨

    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

## 배포 상태 확인 방법

1. GitHub 저장소 페이지 접속
2. 상단 `Actions` 탭 클릭
3. 가장 최근 워크플로우 실행 결과 확인

| 아이콘 | 상태 |
|--------|------|
| 노란색 원 (●) | 실행 중 |
| 초록색 체크 (✓) | 성공 — 배포 완료 |
| 빨간색 X (✗) | 실패 — 로그 확인 필요 |

---

## 수동으로 배포 실행하는 방법

코드 변경 없이 다시 배포하고 싶을 때:

1. GitHub 저장소 → `Actions` 탭
2. 왼쪽 목록에서 `Deploy Hugo site to GitHub Pages` 클릭
3. 오른쪽 `Run workflow` 버튼 클릭
4. `Run workflow` 확인 클릭
