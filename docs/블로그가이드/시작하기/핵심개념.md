# 핵심 개념

## 1. GitHub Pages

GitHub는 코드를 저장하는 서비스입니다.  
GitHub Pages는 GitHub가 제공하는 **무료 웹 호스팅** 기능입니다.

- `gjwoo1996.github.io`라는 이름의 저장소(repository)를 만들면  
  → 자동으로 `https://gjwoo1996.github.io` 주소로 웹사이트가 생깁니다.
- 서버 비용이 없습니다. 단, **정적 웹사이트**만 가능합니다.

> **정적 웹사이트란?**  
> 데이터베이스나 서버 코드 없이, HTML/CSS/JS 파일만으로 이루어진 사이트.  
> 블로그, 포트폴리오처럼 내용이 자주 바뀌지 않는 사이트에 적합합니다.

---

## 2. Hugo

글(Markdown)을 HTML로 변환해주는 **정적 사이트 생성기**입니다.

HTML을 직접 작성하지 않아도 됩니다. 글만 쓰면 Hugo가 알아서 웹사이트 형태로 바꿔줍니다.

```
hello-world.md (마크다운, 읽기 쉬운 텍스트)
        ↓  hugo 명령어 실행
hello-world/index.html (브라우저가 읽는 파일)
```

---

## 3. PaperMod

Hugo가 HTML을 만들 때 어떻게 생겼는지 정해주는 **테마(디자인 템플릿)** 입니다.

Hugo 자체는 레이아웃이 없습니다. PaperMod를 설치하면 깔끔한 블로그 디자인이 자동 적용됩니다.

PaperMod가 제공하는 기능:
- 상단 네비게이션 메뉴
- 다크모드 / 라이트모드 자동 전환
- 목차(Table of Contents)
- 검색 기능
- 코드 복사 버튼
- 읽기 시간 표시
- 이전글 / 다음글 버튼

## 4. Git

**코드 변경 이력을 관리하고 GitHub에 업로드하는 도구**입니다.

워드 파일의 "버전 기록" 기능과 비슷하지만, 모든 파일에 적용되고 훨씬 강력합니다.

```bash
git add .                # 변경한 파일을 "저장 예정" 목록에 올리기
git commit -m "글 추가"  # 변경 이력을 로컬에 저장 (스냅샷)
git push                 # GitHub 서버로 업로드
```

---

## 5. GitHub Actions

GitHub에 코드를 올리면 **자동으로 특정 작업을 실행해주는 자동화 도구**입니다.

이 블로그에서는 `.github/workflows/hugo.yml` 파일이 그 역할을 합니다.

```
push(git push) 이벤트 감지
        ↓
Ubuntu 서버 자동 실행
        ↓
Hugo 설치 → 빌드(hugo --minify) → public/ 폴더 생성
        ↓
GitHub Pages에 자동 배포
```

`git push`만 하면 나머지는 GitHub가 알아서 처리합니다.

## 6. Git Submodule

PaperMod 테마는 **다른 사람의 GitHub 저장소**입니다.  
이것을 내 저장소 안에 통째로 복사하지 않고,  
"이 경로에는 저 저장소를 가져다 써"라고 연결만 해두는 방식이 **서브모듈**입니다.

`.gitmodules` 파일에 연결 정보가 저장됩니다:

```toml
[submodule "themes/PaperMod"]
    path = themes/PaperMod
    url = https://github.com/adityatelange/hugo-PaperMod.git
```

**장점:** 테마 원작자가 업데이트하면 아래 명령어로 쉽게 최신 버전을 받을 수 있습니다.

```bash
git submodule update --remote --merge
```
