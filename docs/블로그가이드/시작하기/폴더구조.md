# 폴더 구조

## 전체 디렉토리 구조

```
gjwoo1996.github.io/
│
├── hugo.toml                  ← 블로그 전체 설정 (제목, 메뉴, 기능 ON/OFF)
├── .gitignore                 ← Git이 무시할 파일 목록
├── .gitmodules                ← PaperMod 테마 연결 정보
│
├── .github/
│   └── workflows/
│       └── hugo.yml           ← 자동 배포 스크립트 (건드릴 일 거의 없음)
│
├── content/                   ← ★ 글을 쓰는 곳 (가장 자주 사용)
│   ├── posts/
│   │   └── hello-world.md     ← 실제 블로그 글
│   └── search/
│       └── index.md           ← 검색 페이지 (설정용, 글 아님)
│
├── archetypes/
│   └── default.md             ← 새 글 만들 때 자동으로 채워지는 템플릿
│
├── static/                    ← 파비콘, 이미지 등 그대로 복사되는 파일
├── assets/                    ← Hugo가 처리할 CSS/JS (고급 커스텀용)
├── layouts/                   ← 테마 HTML 덮어쓰기 (고급 커스텀용)
├── i18n/                      ← 언어 번역 파일 (고급 커스텀용)
│
├── themes/
│   └── PaperMod/              ← PaperMod 테마 (절대 직접 수정 금지)
│
└── public/                    ← Hugo 빌드 결과물 (자동 생성, git에 포함 불필요)
```

---

## 폴더별 역할 상세

### `content/` — 가장 자주 사용하는 폴더

모든 블로그 글이 이 폴더 안에 있습니다.

```
content/
├── posts/          ← 블로그 글 (.md 파일)
└── search/
    └── index.md    ← 검색 페이지 (한 번 만들면 수정할 일 없음)
```

새 글을 쓸 때는 `content/posts/` 안에 `.md` 파일을 만들면 됩니다.

---

### `static/` — 이미지, 파비콘 저장 폴더

이 폴더에 넣은 파일은 빌드 시 그대로 루트(`/`)로 복사됩니다.

```
static/
├── favicon.ico           → /favicon.ico 로 접근 가능
└── images/
    └── profile.png       → /images/profile.png 로 접근 가능
```

글에서 이미지를 사용할 때:

```markdown
![프로필 사진](/images/profile.png)
```

---

### `archetypes/` — 새 글 기본 템플릿

`hugo new` 명령어로 글을 생성할 때 이 템플릿이 자동으로 적용됩니다.

---

### `layouts/`, `assets/`, `i18n/` — 테마 커스터마이징

현재는 비어 있습니다. 테마를 직접 수정하고 싶을 때 이 폴더를 사용합니다.  
자세한 내용은 `테마커스터마이징.md` 문서를 참고하세요.

---

### `themes/PaperMod/` — 테마 폴더

**이 폴더 안의 파일은 절대 직접 수정하지 않습니다.**  
Git Submodule로 연결된 외부 저장소이기 때문에, 직접 수정하면 테마 업데이트 시 변경사항이 사라집니다.

---

### `public/` — 빌드 결과물 (git에 포함 금지)

`hugo` 명령어를 실행하면 자동으로 생성됩니다.  
GitHub Actions가 CI 서버에서 직접 생성하므로, 이 폴더를 git에 올릴 필요가 없습니다.  
`.gitignore`에 `public/`이 등록되어 있어야 합니다.
