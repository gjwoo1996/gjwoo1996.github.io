# 전체 개요

## 내 글이 인터넷에 올라가는 과정

GitHub Pages 블로그는 아래 순서로 동작합니다.

```
내 PC에서 글 작성 (Markdown 파일)
        ↓
Hugo가 HTML/CSS로 변환
        ↓
Git으로 GitHub에 업로드
        ↓
GitHub Actions(CI/CD)가 자동 빌드
        ↓
GitHub Pages가 인터넷에 배포
        ↓
브라우저에서 gjwoo1996.github.io 접속 가능
```

## 사용하는 도구 한눈에 보기

| 도구 | 역할 |
|------|------|
| **GitHub Pages** | 무료 웹 호스팅 — 사이트를 인터넷에 공개 |
| **Hugo** | 마크다운 글을 HTML로 변환하는 정적 사이트 생성기 |
| **PaperMod** | Hugo에 적용하는 블로그 디자인 테마 |
| **Git** | 코드 버전 관리 및 GitHub 업로드 도구 |
| **GitHub Actions** | push 이벤트를 감지해 빌드·배포를 자동 실행 |
| **Git Submodule** | PaperMod 테마 저장소를 내 저장소에 연결하는 방식 |

## 전체 흐름 한눈에 정리

```
[내 PC]
  1. hugo new posts/글이름.md      → content/posts/ 에 파일 생성
  2. 마크다운으로 글 작성
  3. hugo server -D                → localhost:1313 에서 미리보기
  4. git add . → git commit → git push

[GitHub]
  5. push 감지 → Actions 자동 실행
  6. Hugo 빌드 → public/ 생성
  7. GitHub Pages에 배포

[인터넷]
  8. https://gjwoo1996.github.io 에서 확인
```

이 흐름을 이해하면 블로그 운영에 필요한 90%는 커버됩니다.
