# 주의사항 및 자주 하는 실수

## 자주 하는 실수 목록

| 실수 | 결과 | 해결 방법 |
|------|------|-----------|
| `draft: true` 상태로 push | 글이 배포 안됨 | `draft: false`로 변경 후 재push |
| 이미지를 `content/` 안에 저장 | 경로가 꼬여 이미지 안 보임 | 이미지는 반드시 `static/images/`에 저장 |
| `themes/PaperMod/` 직접 수정 | 테마 업데이트 시 변경사항 사라짐 | `layouts/`, `assets/`에 오버라이드 파일 생성 |
| `public/` 폴더를 git에 포함 | 저장소 용량 낭비, 빌드 충돌 | 아래 해결 방법 참고 |
| 날짜를 미래로 설정 | 목록에 안 나타날 수 있음 | 현재 날짜 사용 |
| 이미지 파일명에 한글/공백 사용 | URL 인코딩 오류 | 영문 소문자 + 하이픈 사용 |

## public/ 폴더가 git에 포함된 경우 해결 방법

`public/`은 Hugo가 자동 생성하는 빌드 결과물입니다.  
GitHub Actions가 CI 서버에서 매번 새로 만들기 때문에 git에 포함될 필요가 없습니다.

### 확인 방법

```bash
git ls-files public/ | head -5
# 결과가 있으면 tracked 상태
```

### 해결 방법

```bash
# git 추적에서 제거 (파일은 로컬에 남음)
git rm -r --cached public/
git commit -m "chore: remove tracked public/ build artifacts"
git push
```

`.gitignore`에 `public/`이 있는지 확인합니다:

```
public/
.hugo_build.lock
resources/_gen/
```

## Front Matter 날짜 형식

날짜는 아래 형식을 정확히 지켜야 합니다:

```yaml
date: 2026-05-03T10:00:00+09:00
#     ↑날짜    ↑시간    ↑한국시간대 (UTC+9)
```

- 시간이 없으면 `2026-05-03` 형식도 가능합니다.
- **미래 날짜**를 입력하면 `hugo server`에서는 보이지만, 빌드 시 제외될 수 있습니다.

## 로컬에서는 보이는데 배포 후 안 보이는 경우

다음 순서로 확인합니다:

1. **draft 확인**: `draft: false` 인지 확인
2. **날짜 확인**: 날짜가 미래가 아닌지 확인
3. **Actions 확인**: GitHub `Actions` 탭에서 빌드 성공 여부 확인
4. **캐시 문제**: 브라우저 캐시를 지우고 다시 접속 (`Ctrl + Shift + R`)

## hugo.toml 수정 후 변경이 반영 안 되는 경우

로컬 서버를 재시작해야 합니다:

```bash
# Ctrl+C 로 서버 종료 후 다시 실행
hugo server -D
```

## 서브모듈(PaperMod)이 비어있는 경우

저장소를 새로 clone했을 때 `themes/PaperMod/`가 비어있을 수 있습니다.

```bash
# 서브모듈 초기화 및 받아오기
git submodule update --init --recursive
```
